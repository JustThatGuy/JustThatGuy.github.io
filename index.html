Profielwerkstuk

Robin Potze | 2017

Dit is mijn profielwerkstuk, het project dat elke niet technasiumstudent en student met boekhouden moet maken in hun examenjaar. Ik heb het verslag verwerkt in het project zelf, een website met een parallax-scrolling systeem voor het vak informatica. Ik heb voor dit vak gekozen omdat ik een vervolgstudie multimedia design & development wil doen op de Hanzehogeschool. Dit was wel een redelijke voorbereiding.

Plan van aanpak

"Hoe doe ik dit?"

Ik begin met het onderzoeken van html en het leren van de progammeertalen, hierna bedenk ik waar de site over moet gaan en dan verzin ik een mooi ontwerp bij het onderwerp en het doel van de site. Dit ga ik ontwerpen doormiddel van css en jquery. Het verslag schrijf ik nadat de site staat. Voor het onderzoek ga ik op het internet kijken en samenwerken met Jordy van Dijken.

Ik gebruik Visual Studio en Notepad++ als text editors. De site komt op GitHub te staan, zodat de code gemakkelijk kan worden bekeken door anderen. Ook omdat dit geen extra kosten in rekening brengt.

Onderzoek

"Ahhhh oke nu snap ik het"

Ik heb onderzoek gedaan op verschillende, ook in de bron vermelde, sites. Sites zoals w3schools hebben me veel geleerd over hoe ik een website maak qua technische aspecten, waar sites zoals urban-walks en marcin dmoch me inspireerden voor een goed ontwerp.

Door het onderzoek heb ik veel geleerd van parallax en html zelf. Dit moet ook wel via onderzoek, want anders kan je het onmogelijk leren. Het is en blijft een programmeertaal waar je mee werkt. Het verbaaste me hoeveel mogelijkheden voor effecten er zijn.

Uitwerking

"Het werkt niet, wat gaat hier fout?"

Hier de link naar mijn GitHub waar de code te vinden is. De jquery en een klein stukje van de style staan in de index, omdat het ronduit niet werkte in de css en js bestanden. Het is minder mooi, maar het is functioneel.

Evaluatie

"Zo slecht is het niet, toch?"

Ik ben begonnen met het uitzoeken hoe ik een website maak. Ik wist dat het in html gecodeerd moest worden, maar niet dat er ook nog css en andere programmeertalen in html verwerkt konden worden. Toen ik de basis had staan ging ik de slides ontwerpen en als laatste heb ik de jquery toegevoegd, zodat alles mooi bewoog. Ik wilde graag wat meer versieringen toe voegen, maar daar kwam ik niet aan toe.

Ik begon met een idee met een pagina-menu aan de linker kant van het beeldscherm, maar dit plan is mislukt. Dat is hier te vinden.

Ik heb veel tijd aan dit project besteed, maar er ook veel van geleerd. Ik wist vrijwel niets van html toen ik begon, maar ik kan me ondertussen aardig redden door het zoeken op websites gerelateerd aan dit onderwerp. Het besef van hoeveel werk er eigenlijk in een mooie site gaat is er nu ook. Ik heb respect voor mensen die dit als hun vak hebben gekozen. Jordy van Dijken heeft me voornamelijk geholpen met het scrollen naar een pagina als er op een van de linkjes in de header word geklikt.

Ik heb de opdracht voltooid, dus vind ik dat het project een voldoende verdiend. Het is jammer dat ik niet toe kwam aan de extra mogelijkheden, maar daarvoor zal ik meer ervaring op moeten doen. Aangezien de site nogal simpel is zou ik niet hoger geven dan een 7. Het werk verliep soepel en ik kon veel dingen zelf uitzoeken, hierom vind ik het resultaat wel oke.
